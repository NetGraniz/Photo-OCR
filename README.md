# Photo OCR

Photo OCR - статический OCR-инструмент для быстрого распознавания текста с изображений в браузере. Сайт работает на HTML, CSS и JavaScript, не требует backend, базы данных, платных API или API-ключей.

Изображения не загружаются на сервер и не сохраняются. Распознавание выполняется локально в браузере пользователя через [Tesseract.js](https://tesseract.projectnaptha.com/).

## Как пользоваться

1. Сделайте скриншот или скопируйте изображение.
2. Откройте сайт.
3. Нажмите `Ctrl+V`, чтобы вставить изображение из буфера обмена, или выберите файл кнопкой загрузки.
4. Выберите язык OCR: русский, английский или русский + английский.
5. Нажмите `Распознать текст`.
6. Дождитесь завершения OCR.
7. Отредактируйте результат при необходимости.
8. Нажмите `Копировать текст`, чтобы скопировать весь текст из поля результата.
9. Вставьте новое изображение через `Ctrl+V`, чтобы автоматически заменить старое изображение и очистить прежний текст.

## Публикация на GitHub Pages

1. Загрузите файлы проекта в репозиторий GitHub.
2. Откройте `Settings` -> `Pages`.
3. В разделе `Build and deployment` выберите `Deploy from a branch`.
4. Укажите ветку `main` и папку `/root`.
5. Нажмите `Save`.
6. После публикации GitHub покажет ссылку на сайт.

Для репозитория `NetGraniz/Photo-OCR` сайт обычно будет доступен по адресу:

```text
https://netgraniz.github.io/Photo-OCR/
```

## Ограничения Tesseract.js

- Первое распознавание может быть медленным, потому что браузер загружает OCR-движок и языковые модели.
- Качество OCR зависит от резкости, контраста, размера и наклона изображения.
- Рукописный текст, сложные таблицы, мелкий шрифт и декоративные шрифты распознаются хуже.
- Очень большие изображения могут потреблять много памяти и работать медленнее на телефонах.
- Для загрузки Tesseract.js и языковых моделей нужен доступ к CDN при первом запуске.

## Поддерживаемые языки

- Русский: `rus`
- Английский: `eng`
- Русский + Английский: `rus+eng`

## Приватность

Проект не использует backend, не отправляет изображения на сервер, не хранит изображения и не использует внешние OCR API. Вся обработка выполняется в браузере пользователя.
